# OpenClaw Admin 产品需求文档

> 版本：v1.0 | 更新日期：2026-06-29

## 1. 产品概述

### 1.1 产品定位

OpenClaw Admin 是 OpenClaw AI Agent 平台的**本地管理控制台**，面向 OpenClaw 用户和开发者，提供全方位的 Agent 运行监控、能力管理、用量分析和日志查看能力。

### 1.2 目标用户

| 用户角色 | 使用场景 |
|---------|---------|
| OpenClaw 用户 | 监控自己的 Agent 运行状态，查看用量和费用，管理定时任务 |
| OpenClaw 开发者 | 调试 Agent 行为，分析日志，检查配置和能力加载情况 |
| 运维人员 | 监控系统健康度，查看稳定性事件，管理 Cron 任务 |

### 1.3 核心价值

1. **全景可视**：一个界面看清 Agent 的运行状态、能力、用量和日志
2. **零配置启动**：安装即用，自动发现 OpenClaw 数据目录
3. **只读安全**：不修改任何 OpenClaw 数据，放心使用
4. **跨平台**：支持 macOS、Linux、Windows

## 2. 全局功能需求

### 2.1 主题系统

**需求**：支持三种主题模式，满足不同使用环境偏好。

| 模式 | 行为 |
|------|------|
| 日间 | 固定浅色主题 |
| 夜间 | 固定深色主题 |
| 随系统（默认） | 实时跟随操作系统主题切换 |

**交互**：顶栏右侧下拉菜单，三个选项，当前选中项带 ✓ 标记。

### 2.2 数据刷新

**需求**：所有页面数据自动定时刷新，保持信息时效性。

| 页面 | 刷新间隔 |
|------|---------|
| 所有主页面 | 30 秒 |
| 通知中心 | 60 秒 |

**加载状态**：首次加载显示骨架屏，后续刷新静默更新。

**错误处理**：API 请求失败时显示错误信息 + 重试按钮。

### 2.3 通知中心

**需求**：聚合多类系统事件，帮助用户及时发现异常。

**通知源**：
1. Cron 任务执行失败
2. Cron 任务被禁用
3. 稳定性事件（启动失败、崩溃、OOM 等）
4. 配置变更（写入/删除操作）

**交互**：顶栏铃铛图标 + 下拉面板，未读计数角标，点击可跳转到对应页面。

### 2.4 响应式布局

| 屏幕尺寸 | 布局 |
|---------|------|
| 桌面（≥1024px） | TopBar 完整导航 + 内容区最大宽度 1440px |
| 平板（768-1023px） | 左侧图标导航栏（72px）+ TopBar |
| 移动端（<768px） | TopBar 精简 + 底部固定导航栏 |

## 3. 页面需求

### 3.1 仪表盘（`/`）

**目标**：一眼看清系统整体健康状态。

#### 3.1.1 系统状态卡片

展示 4 个核心指标卡片：

| 卡片 | 内容 | 变体 |
|------|------|------|
| Gateway | WebSocket 地址、平台、运行版本 | connected/disconnected |
| Agent | Agent 数量、最新会话活跃时间 | active/idle |
| 能力 | 插件数 + 技能数 + 扩展数 | - |
| 任务 | 总数 / 活跃 / 失败 | healthy/warning/error |

#### 3.1.2 数据源连接状态

展示 11 个数据源的健康度：

| 数据源 | 检测方式 |
|--------|---------|
| 会话索引、记忆库、ChromaDB、配置文件、技能目录、扩展目录、内部日志 | 文件/目录是否存在 |
| Cron 数据库 | SQLite 文件 + cron_jobs 表 |
| 网关日志 | gateway.log 文件 |
| 用量数据 | JSONL 文件 + usage 字段采样 |

**状态**：`connected`（已连接）/ `partial`（部分可用）/ `not_connected`（未连接），未连接的给出修复建议。

#### 3.1.3 Agent/Cron 概览

展示 Agent 列表摘要和 Cron 任务状态分布。

---

### 3.2 Agent 管理（`/agents`）

**目标**：查看所有 Agent 的会话历史和 Token 使用情况。

#### 功能需求

| 功能 | 说明 |
|------|------|
| Agent 列表 | 展示 ID、名称、会话数、最后活跃时间 |
| 会话表格 | 按更新时间排序，展示会话 Key、模型、Token 使用量、状态 |
| 会话展开 | 点击会话可查看详细信息 |
| 搜索 | 按会话 Key 搜索 |
| 类型过滤 | 按 Cron/Discord/Telegram/Main 过滤 |
| Token 进度条 | 可视化 input/output token 比例 |

---

### 3.3 能力中心（`/capabilities`）

**目标**：管理 Agent 的插件、技能和扩展。

#### 3.3.1 插件

**数据来源**：从 `openclaw.json` 的 `plugins.entries` 和 `plugins.allow` 读取，目录扫描补充。

**展示**：名称、启用状态、插槽类型（memory-core/channel/provider/plugin）、描述。

**过滤**：搜索 + 按启用/禁用筛选。

#### 3.3.2 技能

**数据来源**：3 个来源合并去重：
1. `~/.openclaw/skills/`（OpenClaw 管理，24 个）
2. `workspace/skills/`（工作区自有，15 个独立）
3. `skills/` 中的符号链接（个人技能，25 个）

**展示**：名称、描述、Emoji、来源标签（managed/workspace/personal）、可用/禁用状态。

**过滤**：搜索 + 按状态筛选。

#### 3.3.3 扩展

**数据来源**：`~/.openclaw/extensions/` 目录扫描。

**展示**：名称、描述、路径。

---

### 3.4 定时任务（`/cron`）

**目标**：管理 Cron 任务，监控执行健康度。

#### 3.4.1 任务列表

**数据来源**：`openclaw.sqlite` 的 `cron_jobs` 表。

**展示**：任务 ID、名称、Agent、Cron 表达式、下次执行时间、最后执行状态。

#### 3.4.2 健康状态机

每个任务有 5 种健康状态：

| 状态 | 含义 | 颜色 |
|------|------|------|
| scheduled | 已排期，等待执行 | 绿色 |
| due | 已到期，即将执行 | 蓝色 |
| late | 超时未执行 | 红色 |
| unknown | 无法判断 | 灰色 |
| disabled | 已禁用 | 灰色 |

**排序**：late 优先，然后按状态优先级排列。

**统计区**：展示各状态数量分布，late > 0 时红色警告。

#### 3.4.3 执行历史

**数据来源**：`openclaw.sqlite` 的 `cron_run_logs` 表（最近 200 条）。

**展示**：任务 ID、状态（ok/error/timed_out/running）、开始时间、耗时、错误信息。

**过滤**：按状态筛选。

---

### 3.5 用量中心（`/usage`）

**目标**：分析 Token 使用量和费用，支持多维度拆分。

#### 3.5.1 时间段统计

5 个时间段卡片：

| 时间段 | 说明 |
|--------|------|
| 今日 | 00:00 至今 |
| 昨日 | 前一日全天 |
| 近 3 日 | 最近 3 天 |
| 近 7 日 | 最近 7 天 |
| 近 30 日 | 最近 30 天 |

每个卡片展示：Token 总量、费用（¥）、请求数、趋势标签（上升/平稳/下降/无基线）。

#### 3.5.2 多维度拆分

5 个维度的拆分表格：

| 维度 | 说明 |
|------|------|
| 按 Agent | 不同 Agent 的用量分布 |
| 按模型 | mimo-v2.5、MiniMax-M3 等 |
| 按 Provider | Xiaomi、MiniMax、SenseNova 等（已标准化去重） |
| 按会话类型 | Cron / Discord / Telegram / Main |
| 按 Cron 任务 | 不同定时任务的用量 |

每个维度展示：标签、Token 总量、费用、请求数、进度条（相对于总量的百分比）。

#### 3.5.3 数据采集

- **来源**：`.jsonl` 和 `.trajectory.jsonl` 两种会话日志文件
- **去重**：按时间戳秒级 + 模型 + Token 数去重
- **缓存**：服务端预热 + 10 秒后台刷新，首次加载秒级响应
- **精度说明**：与 Provider 后台有差异（子代理/工具重试等中间调用不在日志中）

#### 3.5.4 费用计算

- **有定价模型**：按官方定价表计算人民币费用
- **无定价模型**：回退到 OpenClaw 记录的 USD 成本，按 7.2 汇率换算

### 3.6 记忆库（`/memory`）

**目标**：查看 Agent 的记忆系统状态。

#### 功能需求

| 模块 | 说明 |
|------|------|
| 记忆架构 | workspace/memory/ 目录递归展示，展示目录结构和文件分布 |
| 向量数据库 | ChromaDB 记录总数、分类/来源分布、嵌入状态 |
| 核心文件 | MEMORY.md / SOUL.md / SESSION-STATE.md 等核心记忆文件的状态和大小 |
| 经验积累 | .learnings/ 目录内容 |

---

### 3.7 系统配置（`/config`）

**目标**：查看 OpenClaw 完整配置，无需手动打开 JSON 文件。

#### 功能需求

| 功能 | 说明 |
|------|------|
| 分区展示 | 按配置域分区（agents、plugins、model、connectors 等） |
| JSON 树视图 | 递归展示完整配置树，支持展开/收起 |
| 敏感字段脱敏 | 匹配 `key|secret|token|password` 的字段自动 mask，点击眼睛图标切换显示 |
| `$VAR` 引用解析 | 自动解析配置中的环境变量引用，展示真实值 |
| 只读 | 不支持修改配置 |

---

### 3.8 日志中心（`/logs`）

**目标**：统一查看所有类型的系统日志，支持结构化展示和快速定位。

#### 3.8.1 日志类型

| Tab | 数据源 | 说明 |
|-----|--------|------|
| 网关日志 | `gateway.log` | OpenClaw 网关运行日志 |
| 命令日志 | `commands.log` | Agent 命令执行记录 |
| 错误日志 | `gateway.err.log` + `gateway.log` 过滤 | warn/error 级别日志，合并新旧来源 |
| 重启日志 | `gateway-restart.log` + `gateway.log` 过滤 | 网关重启事件，合并新旧来源 |
| 配置审计 | `config-audit.jsonl` | 配置变更记录 |
| 稳定性事件 | `stability/*.json` | 启动失败、崩溃、OOM 等事件 |
| 文件列表 | `logs/` 目录 | 所有日志文件元信息 |

#### 3.8.2 结构化展示

所有日志行统一为结构化卡片展示：

| 列 | 内容 |
|----|------|
| 状态标识 | 彩色圆点（info 绿 / warn 黄 / error 红） |
| 时间 | 本地时间格式 |
| 来源 | 组件 Badge（网关/工具/命令/记忆等） |
| 消息摘要 | 本地化关键信息 |
| 标签 | Key-Value 高亮（model、sessionKey 等） |

**展开功能**：点击任意行展开查看完整原始内容。

#### 3.8.3 特殊处理

- **多行合并**：错误日志中的多行堆栈信息合并为单条记录
- **重复去重**：相同模式的警告合并为聚合条目（显示 "x N" 和涉及的技能列表）
- **来源合并**：错误日志和重启日志自动合并新旧两个来源，按时间倒序排列

## 4. 非功能需求

### 4.1 性能

| 指标 | 目标 |
|------|------|
| 首次加载（仪表盘） | < 1 秒 |
| 用量中心首次加载 | < 500ms（预热缓存） |
| 页面切换 | 即时（SPA 路由） |
| 数据刷新延迟 | ≤ 30 秒（轮询间隔） |

### 4.2 兼容性

| 平台 | 支持度 |
|------|--------|
| macOS | 完整支持（推荐） |
| Linux | 完整支持（需安装 sqlite3） |
| Windows | 完整支持（需安装 sqlite3） |

### 4.3 安全性

| 需求 | 实现 |
|------|------|
| 敏感字段脱敏 | 配置页自动识别并 mask，支持手动切换 |
| 只读模式 | 所有页面均为只读，不修改任何 OpenClaw 数据 |
| 无网络暴露 | 默认仅监听 localhost，远程访问需手动 `--host 0.0.0.0` |

### 4.4 可部署性

| 需求 | 说明 |
|------|------|
| 环境变量配置 | `OPENCLAW_HOME`、`OPENCLAW_SYS_LOGS_DIR` 可覆盖所有路径 |
| 零配置启动 | `npm install && npm run dev` 即可使用 |
| 跨平台路径 | 自动适配 macOS/Linux/Windows 的目录结构 |

## 6. 术语表

| 术语 | 说明 |
|------|------|
| OpenClaw | AI Agent 平台，本项目的管理对象 |
| Agent | OpenClaw 中的 AI 代理实例 |
| Session | Agent 的一次对话会话 |
| Plugin | OpenClaw 的功能插件（渠道、模型提供者、记忆核心等） |
| Skill | Agent 的技能（可被调用的工具/能力） |
| Extension | Agent 的扩展包（可包含多个 Skill） |
| Cron Job | 定时任务，按 Cron 表达式定期触发 Agent 执行 |
| ChromaDB | 向量数据库，用于存储 Agent 的语义记忆 |
| Trajectory | Agent 的运行轨迹日志（`.trajectory.jsonl`），包含完整的 API 调用记录 |
| Provider | 大模型提供者（Xiaomi、MiniMax、OpenAI 等） |
| Token | 大模型的计量单位，分为 input（输入）、output（输出）、cacheRead（缓存命中） |
